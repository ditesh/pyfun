A side trek to compilers (deeper the rabbit hole we go)

Compilers basically do the following:

  1. Read in your code
  2. Do a grammar check (also known as linting)
  3. Convert the code to an Abstract Syntax Tree (trees are very common structures in CS)
  4. Do a semantic check (type checking happens here)
  5. Repeated optimization via tree manipulation (super fun)
  6. Walk the tree and output in a different language (memory allocation happens here)

Euclid's Algorithm

\[\begin{aligned} a & = n_1 f \\ b & = n_2 f \\ a - b & = (n_1 - n_2) f \\ \end{aligned} \]
\[\begin{aligned} a \mod b & = a - kb \\ & = n_1 f - k n_2 f \\ & = ( n_1 - k n_2 ) f \end{aligned} \]

GCD Algorithm In Code


def gcd(a, b):
    while b != 0:
        copyOfB = b

        # % is the modulus operator
        b = a % b
        a = copyOfB

    return a
