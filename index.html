Compilers basically do the following:

  1. Read in your code

  4. Do a semantic check (type checking happens here)
  5. Repeated optimization via tree manipulation (super fun)
  6. Walk the tree and output in a different language (memory allocation happens here)

Euclid's Algorithm

\[\begin{aligned} a & = n_1 f \\ b & = n_2 f \\ a - b & = (n_1 - n_2) f \\ \end{aligned} \]
\[\begin{aligned} a \mod b & = a - kb \\ & = n_1 f - k n_2 f \\ & = ( n_1 - k n_2 ) f \end{aligned} \]

def gcd(a, b):
    while b != 0:
        copyOfB = b

        # % is the modulus operator
        b = a % b
        a = copyOfB

    return a

def fib():

    data = {}

    def helper(n):

        if n not in data:
            data[n-2] = fib(n-2)
            data[n-1] = fib(n-1)
            data[n] = data[n-2] + data[n-1]

        return data[n]

    # we can return a function
    return helper

fn = fib()
print fn(30)

# vals here is known as a variadic arg
# it is passed in as a list
def sums(*vals):

    retval = 0
    for i in vals: retval += i
    return retval

print sums(1,10,-300)
